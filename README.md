Comecei com um for simples. Acabei com 30 linhas de lógica que eu nem planejei escrever.
Tava só testando como o for i in range() funcionava na prática. Sem pretensão nenhuma.
Aí printei a tabuada de um número, funcionou, e aí veio aquele pensamento clássico:
"e se eu deixar o usuário escolher o número?"
Aí virou input. Aí pensei: "e se ele digitar uma letra?" Virou try/except. "E se ele quiser calcular outro número?" Virou while. "E se ele responder diferente de Y ou N?" Virou validação dentro de validação.
Uma coisa foi puxando a outra, completamente orgânico, sem tutorial, sem seguir nenhum roteiro.
E foi aí que entendi como programação funciona de verdade. Não é sentar com um plano perfeito. É começar com qualquer coisa pequena e deixar as perguntas te guiarem.
O projeto mais simples que você puder imaginar já é suficiente pra começar. O resto aparece no caminho.
